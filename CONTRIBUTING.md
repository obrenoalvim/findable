# Contributing

The `SKILL.md` is the source of truth. If you find a gap in the rules, a best practice that should be covered, or a behavior that's wrong — open a PR against `skills/findable/SKILL.md`.

## What to contribute

- New SEO/GEO areas the skill doesn't cover yet
- Better research queries
- New safe-vs-sensitive distinctions
- Platform-specific optimizations (Next.js, Django, etc.)
- Source links for existing rules that lack citations

## How

1. Fork the repo
2. Edit `skills/findable/SKILL.md`
3. Open a PR with a link to the source or research behind the change
